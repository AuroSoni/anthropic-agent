"text"

BetaTextBlock / BetaTextBlockParam
Round-trip: Send back exactly as received. The SDK response includes citations as parsed pydantic objects; the API wire format expects them as raw dicts. cache_control is param-only (never returned).

{
  type:      Literal["text"],
  text:      str,
  citations: Optional[List[BetaTextCitation]]  # = None
}

// BetaTextCitation = Union of:
//   BetaCitationCharLocation
//     { type: "char_location",
//       cited_text: str, document_index: int,
//       start_char_index: int, end_char_index: int,
//       document_title: Optional[str],
//       file_id: Optional[str] }
//   BetaCitationPageLocation
//     { type: "page_location",
//       cited_text: str, document_index: int,
//       start_page_number: int, end_page_number: int,
//       document_title: Optional[str],
//       file_id: Optional[str] }
//   BetaCitationContentBlockLocation
//     { type: "content_block_location",
//       cited_text: str, document_index: int,
//       start_block_index: int, end_block_index: int,
//       document_title: Optional[str],
//       file_id: Optional[str] }
//   BetaCitationsWebSearchResultLocation

"tool_result"

BetaToolResultBlockParam (input-only)
Input-only: Never returned by the API. You construct this to send back the result of a tool_use block. content can be a plain string or a list of rich content blocks.

Allowed content block types inside tool_result: text (most common), image (return images from tool execution), document (return documents), search_result (return structured search results with citation support), and tool_reference (associate this result with a specific tool by name — used in multi-tool scenarios, particularly with deferred tools from tool search; see the tool_search_tool_result section for the full flow). 
{
  type:          Required[Literal["tool_result"]],
  tool_use_id:   Required[str],
  content:       Optional[Union[str, Iterable[ContentUnion]]],
  is_error:      Optional[bool],
  cache_control: Optional[CacheControlParam]
}

// ContentUnion = Union of:
//   BetaTextBlockParam
//   BetaImageBlockParam
//   BetaSearchResultBlockParam
//   BetaRequestDocumentBlockParam
//   BetaToolReferenceBlockParam
//     { type: Required["tool_reference"],
//       tool_name: Required[str],
//       cache_control: Optional[CacheControlParam] }

"tool_search_tool_result"

BetaToolSearchToolResultBlock / …Param
Round-trip: Result of a tool search server tool invocation. Used with the tool_search_tool_regex / tool_search_tool_bm25 server tools, which allow the model to search through registered tools instead of loading all tools into the system prompt at once.

Tool Search Flow: 1. Register tools with defer_loading: true — these tools are not included in the initial system prompt. 2. The model invokes a server_tool_use with name: "tool_search_tool_regex" (or bm25). 3. The API returns this tool_search_tool_result block, whose success content is a list of tool_reference items — each one is simply {"type": "tool_reference", "tool_name": "my_deferred_tool"}, telling the model which deferred tools matched the search. 4. The model can now call the matched deferred tools via normal tool_use blocks.

About tool_reference: A tool_reference is a minimal pointer to a tool by name. It appears here in tool search results, and can also appear inside tool_result content to associate a tool result with a specific tool in multi-tool scenarios. Its structure is just {"type": "tool_reference", "tool_name": str}.

SDK Response

{
  type:        Literal["tool_search_tool_result"],
  tool_use_id: str,
  content:     Union[
    BetaToolSearchToolResultError,
    BetaToolSearchToolSearchResultBlock
  ]
}

// Error:
//   { type: "tool_search_tool_result_error",
//     error_code: "invalid_tool_input"
//       | "unavailable" | "too_many_requests"
//       | "execution_time_exceeded",
//     error_message: Optional[str] }
//
// Success:
//   { type: "tool_search_tool_search_result",
//     tool_references: List[
//       { type: "tool_reference",
//         tool_name: str }] }

BetaSearchResultBlockParam (input-only)
Input-only: Never returned by the API. Used to pass your own external search results (e.g. from your own RAG pipeline, Elasticsearch, vector DB, etc.) to the model as structured input.

Not the same as web_search_tool_result — that is a server-executed web search performed by Anthropic's servers. search_result is for your search results.

Why use it? Instead of dumping search results as raw text into a user message, wrapping them in search_result blocks lets the model: (a) understand they are structured search results with a source URL and title, and (b) optionally generate citations pointing back to them. When citations: {"enabled": true} is set and the model references content from these blocks, the response text blocks will contain BetaCitationSearchResultLocation citations (type "search_result_location") pointing back to the specific search_result block by index. 
{
  type:          Required[Literal["search_result"]],
  source:        Required[str],         // URL or source identifier
  title:         Required[str],
  content:       Required[Iterable[BetaTextBlockParam]],
  citations:     Optional[BetaCitationsConfigParam],
  cache_control: Optional[CacheControlParam]
}

Shared Type: tool_reference

BetaToolReferenceBlock / BetaToolReferenceBlockParam
Not a top-level content block. Appears in two contexts:

1. Inside tool_search_tool_result responses — the API returns a list of tool_reference items indicating which deferred tools matched a tool search query. This is part of the tool search feature that allows tools to be registered with defer_loading: true so they aren't loaded into the system prompt until the model searches for and finds them. See tool_search_tool_result for the full flow.

2. Inside tool_result content — you can include tool_reference blocks alongside text/image/document blocks in a tool_result's content array to associate the result with a specific tool by name.

SDK Response — BetaToolReferenceBlock

{
  type:      Literal["tool_reference"],
  tool_name: str
}

API Wire Param — BetaToolReferenceBlockParam

{
  type:          Required[Literal["tool_reference"]],
  tool_name:     Required[str],
  cache_control: Optional[CacheControlParam]
}

Shared Type: Caller

discriminated union on "type"
Appears on tool_use, server_tool_use, web_search_tool_result, and web_fetch_tool_result. Indicates who initiated the tool call. The response and param shapes are identical.

Shared Type: cache_control

BetaCacheControlEphemeralParam (param-only)
Param-only: Never returned by the API. Added by the client to request caching on specific content blocks. Anthropic limits cache_control to 4 blocks maximum per request. Not supported on thinking or redacted_thinking blocks. 
{
  type: Required[Literal["ephemeral"]],
  ttl:  Optional[Literal["5m", "1h"]]  // defaults to "5m"
